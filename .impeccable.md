# Impeccable — Project Design Context

This file is the source of truth for design decisions on swpatterns.com. The `/impeccable` skills read it before doing any design work. Update it when the direction changes — don't let it drift.

---

## Design Context

### Users

The primary reader is a **working software engineer in lookup mode**. They arrived from a search engine with a specific pattern in mind (e.g. "saga orchestration", "circuit breaker"), they have a language they're working in, and they want two things in this order:

1. A short, authoritative definition that confirms they've found the right pattern.
2. A code sample in their language that they can read or copy.

A secondary audience exists — students browsing categories, practitioners cross-referencing related patterns — but every design decision is judged against the lookup case first. Anything that helps the lookup case (clearer hierarchy, faster language switching, scannable metadata) helps the others. Anything that gets in its way (illustrations, tutorial chrome, marketing voice) hurts everyone.

The site is content. It is not an app. Read at a desk, on a laptop, occasionally on a phone while pairing.

### Brand Personality

Three words: **archival, disciplined, Italian-editorial.**

The site projects the voice of a printed reference catalog — the kind of book whose page numbers and section marks are typeset with as much care as its body copy. Authoritative without being academic. Dense without being cluttered. Italian without being kitschy (no flags, no tricolore stripes — Italian as in Bodoni, Vignelli, Domus, Olivetti; not as in pizza emojis).

Emotional goal: when a reader lands on a pattern page, they should feel they've arrived at something **considered**, not generated. The footer already says "Made with ❤️ in Bologna" — the design should earn that line.

### Aesthetic Direction

**Reference**: Italian editorial / Bauhaus archival. Specifically the visual language of Domus, Casabella, Lars Müller monographs, and the Olivetti/Vignelli school of catalog design. Page-as-artifact discipline. Pattern entries are typeset like museum or library catalog cards: numbered, classified, deliberately set.

**Anti-references** (all four were called out — treat as hard exclusions):

- **No Refactoring.guru aesthetic**: no cartoon mascots, no color-coded category pills, no illustrated explainer tone.
- **No generic Hugo/dev blog template**: no centered hero with a stock sans, no identical card grids, no drop-shadow rounded rectangles. The current stock-theme look is exactly what to leave behind.
- **No AI-startup landing page**: no dark + cyan/purple gradients, no glassmorphism, no glowing borders, no gradient text.
- **No corporate enterprise docs**: no navy + battleship-gray IBM/SAP chrome, no dense nav rails that feel legal.

**Theme**: follow system preference (`prefers-color-scheme`) with a manual override toggle preserved. Light is **warm off-white** (book paper, not stock white). Dark is **deep ink** (not pure black; slight cool tint). Both must be designed to the same standard — no "we built it light and inverted it for dark."

**Color**: brand assets are open — drop the stock `#0077b8` navy. Replace with a palette built around an **Italian editorial vermillion** (Domus/Casabella red lineage) as the single saturated accent, on warm-tinted neutrals. Concrete starting palette in OKLCH:

- `--ink`: `oklch(18% 0.012 280)` — body text on light, surface on dark
- `--paper`: `oklch(97% 0.006 75)` — page surface on light
- `--paper-dim`: `oklch(94% 0.008 75)` — secondary surface on light
- `--ink-paper`: `oklch(96% 0.008 75)` — body text on dark
- `--rule`: `oklch(20% 0.012 280 / 0.18)` — hairline rules and dividers
- `--rosso`: `oklch(54% 0.20 25)` — the single accent; used rarely, with intent

Neutrals are tinted toward `~75°` (warm) at very low chroma — never pure gray. The accent appears as: catalog entry numbers, the section mark on the active language tab, the "Nº" prefix, and link underlines on hover. **Not** as a card stripe, not as a button fill on every CTA, not as gradient endpoints. Rare = powerful.

**Typography** (avoid every font in the reflex-reject list):

- **Display / numerals / catalog marks** → **Bodoni Moda** (variable, Google Fonts). Italian heritage — Giambattista Bodoni was the Parmesan punch-cutter who defined modern editorial type. High-contrast, decisive, exactly the cover-of-Domus energy this calls for. Use it for `h1` pattern titles, catalog numbers (`Nº 042`), and section marks. Optical size axis lets it work at both display and metadata sizes.
- **Body / running text / `h2`–`h4`** → **Spectral** (Production Type, free via Google Fonts). Transitional serif designed for screens, with Italian-editorial italics. Workhorse for prose. Pair with Bodoni Moda by letting Bodoni shout and Spectral speak.
- **Code / metadata labels / small caps** → **Cutive Mono** (Vernon Adams, free via Google Fonts). A mid-20th-century typewriter face — evokes the typed card-catalog labels the rest of the design system is modeled on. Substituted from the originally-spec'd Departure Mono because Cutive is reliably available via Google Fonts and is a *better* fit for the archival brief (card-catalog cards were typed, not pixel-rendered). Used for code blocks, language tags, taxonomy labels, and small-cap section markers. **Departure Mono** remains an acceptable substitute if a future contributor wants to self-host it; **JetBrains Mono / Plex Mono are explicitly off-limits**.

If any of these proves impractical (file size, rendering, licensing), the substitution rule is: pick something that holds the same role and **is not** in the reflex-reject list. Don't fall back to Inter, Plex, Space Grotesk, or any of their cousins.

**Layout**: strong asymmetric grid. Left-aligned body, generous outer margins, distinct columns for content vs. metadata sidebar. Pattern pages should read as numbered catalog entries — entry number top-left in Bodoni small-caps, taxonomy classification (`Creational · GoF`) in Departure Mono small caps, title in Bodoni display, body in Spectral. Per-language code samples below, with a language switcher that is the **most prominent interactive element on the page** — that's why most readers came.

No card grids on the homepage. The pattern index is a typographic list — like a printed contents page — not a sea of identical tiles.

### Design Principles

1. **Catalog, not blog.** Every page is a numbered entry in an archive. Metadata (`pattern_types`, `language`, taxonomy) is composed into the design, not bolted on as chrome. Pattern types behave like a classification system, not tags.
2. **Lookup over discovery.** Assume the reader knows what they came for. Title, classification, and language switcher must all be visible without scrolling on a 13" laptop. Related-pattern navigation lives below the fold.
3. **Italian editorial restraint.** High-contrast type, warm neutrals, one rare accent. The design earns its character through typography and grid discipline — not through color, illustration, or decorative shapes.
4. **Code samples are the product.** Per-language code is what brought most readers here. Syntax highlighting must hit WCAG AA on both themes (verify, don't assume). Copy buttons must be keyboard-reachable. Code blocks must be screen-reader friendly (proper `<pre><code>` semantics, language declared).
5. **Dual-mode parity.** Light and dark are co-equal. Accent contrast, syntax highlighting contrast, and rule-line visibility are tuned independently for each. Test both before shipping any layout work.

### Constraints

- **Stack**: Hugo (no JS framework, no build pipeline beyond Hugo + theme SASS). Any new typography or color decisions land in `themes/hugo-clarity-new/assets/sass/` or in `layouts/` overrides.
- **A11y**: WCAG **AA** baseline, plus the extra-care clauses above for code samples (AA syntax contrast, keyboard-reachable copy, screen-reader-friendly code blocks). Respect `prefers-reduced-motion`.
- **Performance**: no JS frameworks, no large icon fonts. Web fonts loaded with `font-display: swap`. Variable fonts preferred (Bodoni Moda is variable; ship one file with multiple weights).
- **Identity**: the "Made in Bologna" footer line is part of the brand — don't strip it in a redesign.
